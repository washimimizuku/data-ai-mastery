# Contributing to Data Engineering & AI Portfolio

Thank you for your interest in contributing! This project aims to help people learn data engineering and AI through structured bootcamps and hands-on projects.

## How to Contribute

### Reporting Issues

**Found a bug or error?**
- Check if the issue already exists
- Open a new issue with a clear title and description
- Include steps to reproduce if applicable
- Specify which bootcamp or project it relates to

**Have a suggestion?**
- Open an issue with the "enhancement" label
- Describe the improvement clearly
- Explain why it would be valuable

### Improving Documentation

**Fix typos or errors:**
- Fork the repository
- Make your changes
- Submit a pull request with a clear description

**Add clarifications:**
- If something was confusing, help make it clearer
- Add examples or explanations
- Update outdated information

### Contributing Content

**Bootcamp improvements:**
- Suggest additional topics or exercises
- Improve existing explanations
- Add practical examples

**Project ideas:**
- Propose new quick win projects
- Suggest comprehensive project topics
- Share implementation insights

### Sharing Your Experience

**Success stories:**
- Share your learning journey
- Describe projects you built
- Mention job outcomes (optional)

**Feedback:**
- What worked well?
- What was challenging?
- What could be improved?

## Pull Request Process

1. **Fork** the repository
2. **Create** a branch (`git checkout -b feature/improvement-name`)
3. **Make** your changes
4. **Test** that links and formatting work
5. **Commit** with a clear message
6. **Push** to your fork
7. **Submit** a pull request

### Pull Request Guidelines

- Keep changes focused and atomic
- Write clear commit messages
- Update relevant documentation
- Ensure markdown formatting is correct
- Reference related issues if applicable

## Code of Conduct

### Our Standards

**Positive behavior:**
- Be respectful and inclusive
- Welcome newcomers
- Give constructive feedback
- Focus on learning and growth

**Unacceptable behavior:**
- Harassment or discrimination
- Trolling or insulting comments
- Personal attacks
- Spam or self-promotion

### Enforcement

Violations may result in:
- Warning
- Temporary ban
- Permanent ban

Report issues to: [repository owner]

## Questions?

- Open an issue with the "question" label
- Check the FAQ.md first
- Be specific about what you need help with

## Recognition

Contributors will be acknowledged in:
- CHANGELOG.md for significant contributions
- README.md contributors section (if added)
